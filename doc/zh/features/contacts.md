# 联系人

联系人是 Bonds 的核心实体。每个联系人都属于一个 [Vault](/zh/features/vaults)，可以存储关于某个人的丰富结构化信息。

## 联系人信息

每个联系人支持：

- **姓名** — 名、姓、昵称、曾用名
- **联系方式** — 邮箱、电话、社交媒体链接（12 种内置类型）
- **地址** — 多个地址，支持类型标记（家庭、工作等），可选地理编码
- **公司** — 职位和公司关联
- **性别与代词** — 可自定义的性别和代词选项
- **宗教** — 可选的宗教信仰

## 模块

联系人详情页由**模块**组成 — 显示在模板页面上的可配置组件。默认模块包括：

| 模块 | 说明 |
|------|------|
| 联系人姓名 | 姓名字段和昵称 |
| 重要日期 | 生日、纪念日、自定义日期 |
| 关系 | 家人、伴侣、朋友 |
| 笔记 | 附加到联系人的自由文本 |
| 任务 | 关联到联系人的待办事项 |
| 提醒 | 定时通知 |
| 通话 | 电话记录及备注 |
| 礼物 | 礼物想法和追踪 |
| 债务 / 借贷 | 借入或借出的金额 |
| 活动 | 共同参与的活动和事件 |
| 人生事件 | 重要里程碑（毕业、结婚等） |
| 宠物 | 宠物名称和类别 |
| 分组 | 联系人分组归属 |
| 文档 | 上传的文件（PDF、图片） |
| 照片 | 照片画廊 |
| 日记 | 带模板的日记式条目 |
| 目标 | 个人目标追踪 |
| 动态 | 活动时间线 |

## 模板

模板控制联系人详情页的布局。每个模板有多个**页面**（标签页），每个页面显示一组模块。默认模板包含：

1. **联系信息** — 头像、姓名、重要日期、性别、标签、公司、宗教
2. **动态** — 活动时间线
3. **社交** — 关系、宠物、分组、地址、联系方式
4. **生活与目标** — 人生事件、目标
5. **信息** — 文档、照片、笔记、提醒、借贷、任务、通话、日记

模板和模块分配可通过[个性化设置](/zh/features/admin#个性化设置)自定义。

## 标签

标签是你可以分配给联系人的标记，用于组织和筛选。创建自定义标签，按你喜欢的方式分类联系人。

## 头像

每个联系人都有头像。如果没有上传照片，Bonds 会自动生成**首字母头像** — 一个带有联系人首字母的彩色圆形。颜色由名字的 MD5 哈希确定性生成，同一个名字始终得到相同的颜色。

## 关系

定义联系人之间的关系 — 父母、子女、伴侣、朋友、同事等。关系类型按组分类：

- **爱情** — 伴侣、配偶、恋人
- **家庭** — 父母、子女、兄弟姐妹
- **友谊** — 密友、熟人
- **工作** — 同事、导师、上司

### 跨 Vault 关系

关系可以跨越不同的 Vault。添加关系时，联系人选择器会显示你有权访问的**所有 Vault** 中的联系人，按 Vault 名称分组。

- 若你对目标 Vault 拥有**编辑者**权限，系统会自动创建**双向关系**（两个联系人都能看到该关系）
- 若你只有**查看者**权限，则只创建**单向关系**，界面会提示原因
- 删除跨 Vault 关系时，会自动清理对方的反向记录
