# 多 Vault

Vault 是存放联系人及其关联数据的隔离容器。它为多用户场景提供数据隔离和访问控制。

## 为什么需要 Vault？

- **数据隔离** — 不同 Vault 中的联系人完全独立
- **共享访问** — 邀请其他用户协作管理同一个 Vault
- **基于角色的权限** — 控制谁能做什么

## 角色

每个用户在 Vault 中拥有以下角色之一：

| 角色 | 权限 |
|------|------|
| **管理者** | 完全访问：创建、编辑、删除联系人及 Vault 设置，可邀请他人 |
| **编辑者** | 可创建和编辑联系人，但不能管理 Vault 设置或用户 |
| **查看者** | 只读访问联系人，无法修改任何内容 |

## 创建 Vault

登录后，从仪表盘创建你的第一个 Vault。为它取个名字并添加可选描述。创建者自动成为**管理者**。

## Vault 级设置

每个 Vault 有自己的默认设置，在创建时自动初始化：

- **重要日期类型** — 生日、逝世日期（内置），可添加自定义类型
- **情绪追踪参数** — 5 级情绪量表，带 emoji 和颜色
- **人生事件分类** — 4 个类别，20 种事件类型
- **快速信息模板** — 爱好、饮食偏好

## 邀请用户

Vault 管理者可以通过[用户邀请](/zh/features/more#用户邀请)系统邀请其他用户加入 Vault，每个邀请指定一个权限级别。

## 切换 Vault

如果你有多个 Vault 的访问权限，使用导航栏中的 Vault 切换器在它们之间切换。每个 Vault 维护自己的联系人列表、设置和搜索索引。
