# 多 Vault

Vault 是存放联系人及其关联数据的隔离容器。它为多用户场景提供安全的数据隔离和访问控制。

## 为什么需要 Vault？

- **数据隔离**：不同 Vault 中的联系人相互独立，但支持[跨 Vault 关系](/zh/features/contacts#跨-vault-关系)。
- **共享访问**：邀请其他用户协作管理同一个 Vault。
- **基于角色的权限**：控制谁能做什么。

## 角色

每个用户在 Vault 中拥有以下角色之一：

| 角色 | 权限 |
|------|------|
| **管理者** | 完全访问：创建、编辑、删除联系人及 Vault 设置，可邀请他人。 |
| **编辑者** | 可创建和编辑联系人，但不能管理 Vault 设置或用户。 |
| **查看者** | 只读访问联系人，无法修改任何内容。 |

## 创建 Vault

登录后，从仪表盘创建你的第一个 Vault。为它取个名字并添加可选描述。创建者自动成为**管理者**。

## Vault 级设置

每个 Vault 有自己的默认设置，在创建时自动初始化：

- **重要日期类型**：生日、逝世日期（内置），可添加自定义类型。
- **情绪追踪参数**：5 级情绪量表，带 emoji 和颜色。
- **人生事件分类**：4 个类别，20 种事件类型。
- **快速信息模板**：爱好、饮食偏好。

## 用户影子联系人（Shadow Contact）

Bonds 引入了影子联系人机制。每个用户在加入每个 Vault 时，系统会在该 Vault 内自动为其生成一个专属的「影子联系人」。
- **关联方式**：影子联系人 ID 保存于 `UserVault.ContactID` 字段，并在 API 中作为 `user_contact_id` 返回给前端。
- **个人数据载体**：此影子联系人用来记录用户个人的心情、人生事件等数据，与外部联系人信息相互隔离。
- **隐私隔离**：影子联系人不会显示在联系人列表、搜索结果或导出的 vCard 中，不可被删除。

## Vault 仪表盘

Bonds 仪表盘采用响应式三栏式布局，组织了工作流中的关键信息：

### 左栏
展示**最近联系人**与**最常查阅**的联系人，供快速跳转。在小尺寸平板设备上会自动隐藏。

### 中栏
拥有一个分段切换组件，提供三个核心标签页。你最后一次选中的标签页会被持久化保存，下次进入时将自动显示该页。
1. **活动动态**：展示该 Vault 内所有联系人数据的变更动态日志。
2. **个人生活事件**：展示关联到你影子联系人下的个人生活大事件与时间线。
3. **生活指标**：轻量级事件计数器。点击「+1」即可记录一次打卡，进入详情页可查看月度打卡柱状图。

### 右栏
包含以下交互组件：
- **心情记录**：快速录入你当天的状态，关联到你的影子联系人。
- **即将到来的提醒**：展示即将来临的通知提醒。
- **到期任务**：展示需要处理的待办事项。

## 生活指标（Life Metrics）架构

生活指标采用「事件日志」模式。每次点击「+1」，系统便会在数据库中记录一条带时间戳的增量事件，而不是修改联系人身上的某个属性值。详情页的月度柱状图正是通过统计这些历史事件日志计算得来的。

## 邀请用户

Vault 管理者可以通过用户邀请系统邀请其他用户加入 Vault，每个邀请指定一个权限级别。

## 切换 Vault

如果你有多个 Vault 的访问权限，使用导航栏中的 Vault 切换器在它们之间切换。每个 Vault 维护自己的联系人列表、设置和搜索索引。
