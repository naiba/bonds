# 提醒

Bonds 内置提醒系统，通过邮件或基于 Shoutrrr 的渠道（如 Telegram）通知你重要的日期和事件。

## 类型

| 类型 | 行为 |
|------|------|
| **一次性** | 在预定日期/时间触发一次，然后完成 |
| **每周** | 每周重复 |
| **每月** | 每月重复 |
| **每年** | 每年重复（适合生日） |

## 工作原理

1. **创建提醒** — 在联系人上设置日期、时间、类型和标签
2. **Cron 调度器** — 每分钟运行一次，扫描到期的提醒
3. **发送通知** — 按每个通知渠道设置的首选发送时间发送
4. **周期性提醒** — 自动基于上次计划时间调度下一次（非当前时间，防止漂移）

## 通知渠道

Bonds 支持邮件以及兼容 Shoutrrr 的通知渠道：

### 邮件

在管理面板中配置 SMTP 设置。用户注册时会自动创建一个以用户邮箱为地址的邮件通知渠道。

### Shoutrrr / Telegram

在用户通知设置中添加 Shoutrrr URL，例如 Telegram URL（`telegram://token@telegram?channels=123456`）。Shoutrrr 渠道创建后会立即启用，并可使用 Shoutrrr 支持的任意服务。

每个渠道都有**首选发送时间**。新建提醒、已有提醒补排程以及周期性提醒重排都会使用该本地时间。为空或非法时回退到 `09:00`。

详见 [Shoutrrr / Telegram 通知](/zh/features/more#telegram-通知)。

## 渠道可靠性

每个通知渠道会追踪失败次数。如果一个渠道连续失败 **10 次**，将被自动禁用以防止刷屏。修复底层问题后，可在用户设置中手动重新启用。

## 通知历史

每次通知尝试都会记录在 `UserNotificationSent` 中，包括：
- 投递状态（成功/失败）
- 错误信息（如果失败）
- 时间戳
